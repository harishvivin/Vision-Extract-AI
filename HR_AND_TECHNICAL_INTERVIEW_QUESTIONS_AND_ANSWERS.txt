SECTION 1: HR & BEHAVIORAL INTERVIEW QUESTIONS & MODEL ANSWERS
====================================================================================================

Q1: "Can you give us a 2-minute introduction to your project, Vision Extract AI?"
----------------------------------------------------------------------------------------------------
ANSWER:
"Vision Extract AI is an intelligent Medical Report PDF Analyzer and Visual Question Answering (QA) Engine. It solves a critical challenge in medical insurance underwriting: manual inspection of multi-page medical reports and pathology lab tests.

When an insurance company receives a 20-page medical examination report—containing identity proofs, complete blood counts, biochemistry, liver function tests, lipid profiles, viral serology, and physician notes—underwriters spend hours searching for specific parameters.

Vision Extract AI automates this workflow:
1. Users upload a PDF medical report. The system renders high-resolution page previews and indexes layout text blocks.
2. Users ask natural language questions (e.g., 'What is the patient's full name?', 'What is the HbA1c percentage?', 'Is the patient diabetic?', or 'Summarize the CBC findings').
3. The AI engine instantly extracts the exact text answer, assigns a confidence score, identifies the exact page number, and auto-crops a high-resolution screenshot snippet of the document highlighting the visual evidence in emerald green.

The project is built with React, Vite, and TailwindCSS on the frontend, powered by a Python FastAPI engine using PyMuPDF and Pillow, and deployed on GitHub Pages with dual-mode browser fallback."


Q2: "What motivated you to build this project?"
----------------------------------------------------------------------------------------------------
ANSWER:
"In healthcare and insurance domains, accuracy and auditability are non-negotiable. Standard text LLMs often hallucinate or provide answers without proof. An insurance underwriter cannot approve a policy based on an unverified text response—they need to see the actual lab snippet with physician signatures and reference ranges.

I wanted to bridge the gap between AI text responses and visual proof. By combining document parsing, intent matching, and dynamic image cropping, I built a system that gives underwriters instant text answers accompanied by pixel-perfect screenshot evidence."


Q3: "What was the biggest technical challenge you faced, and how did you solve it?"
----------------------------------------------------------------------------------------------------
ANSWER:
"The biggest challenge was supporting both local Python backend ML execution and static browser-only deployment on GitHub Pages without breaking the user experience.

On a local server, Python with PyMuPDF and Pillow can dynamically process any PDF and crop image coordinates on the fly. However, GitHub Pages only hosts static files (HTML/JS/CSS).

To solve this, I engineered a dual-mode fallback architecture:
- In full backend mode, requests hit the FastAPI `/api/qa/ask` REST endpoint.
- If the server is offline (e.g., on GitHub Pages), the React app seamlessly transitions to a client-side evaluation engine (`evaluateQueryClientSide`) pre-indexed with 100 ground-truth medical parameters, page previews, and high-resolution snippet crops.

This ensures zero downtime and instant response times whether running locally or on web servers."


Q4: "How did you handle changing user requirements during development?"
----------------------------------------------------------------------------------------------------
ANSWER:
"Initially, the application was a generic PDF page image extractor with diagnostic logs and raw telemetry modals. During iterative reviews, feedback highlighted that users didn't want raw logs or cluttered download buttons; they wanted a clean 2-stage workflow focused solely on asking questions and retrieving screenshot evidence.

I adapted quickly:
- Removed unnecessary telemetry modals and manual download sections.
- Redesigned the UI into a 2-stage workflow: PDF Upload & Analysis -> Active Document Bar + Unlocked QA Evidence Extractor.
- Expanded the QA engine from 6 sample questions to a comprehensive 100 ground-truth dataset covering every pathology test, diagnostic table, and executive summary in the report."

SECTION 2: TECHNICAL ARCHITECTURE & DEEP-DIVE INTERVIEW QUESTIONS
====================================================================================================

Q5: "Walk us through the end-to-end technical pipeline when a user asks a question."
----------------------------------------------------------------------------------------------------
ANSWER:
"1. Input Capture: The user types a question in the React search bar (e.g., 'What is the serum creatinine value?') or clicks a sample prompt chip.
2. API / Fallback Dispatcher: The React component `DocumentQA.jsx` sends a POST request to `/api/qa/ask`.
3. Intent Classification: The Python backend (`DocumentQAEngine`) uses regex pattern matchers trained on 100 medical parameters to classify the question.
4. Parameter Extraction: It maps the intent to answer '0.88 mg/dL.', target Page 13, section title 'Page 13. Serum Creatinine Level', and normalized bounding box `[0.05, 0.25, 0.95, 0.55]`.
5. Visual Evidence Crop: PyMuPDF opens `INPUT_images_and_questions.pdf` at Page 13, renders a 200 DPI bitmap, crops the bounding box with a 15px padding margin, and draws a 5px emerald green outline.
6. JSON Response: Returns JSON with answer text, page number, confidence (99.8%), and image URL.
7. UI Rendering: The React app updates state, displays the answer card with precision badge, and presents the screenshot snippet with full-screen zoom and PNG download options."


Q6: "Why did you choose PyMuPDF (fitz) over traditional Tesseract OCR?"
----------------------------------------------------------------------------------------------------
ANSWER:
"For digital vector PDFs (such as medical lab reports generated directly from LIMS software), PyMuPDF is 50x to 100x faster than traditional OCR engines like Tesseract.

PyMuPDF extracts embedded text objects, font details, and exact bounding box coordinates directly from the PDF stream without requiring expensive CPU/GPU matrix multiplication for character recognition.

However, if an image-only scanned PDF is uploaded, PyMuPDF falls back to rendering page rasters, which can then be fed into Tesseract or OpenCV. Using PyMuPDF as the primary engine gave us sub-second indexing times for 20-page documents."


Q7: "How does the bounding box normalization and image cropping math work?"
----------------------------------------------------------------------------------------------------
ANSWER:
"Bounding box coordinates in PyMuPDF are relative to page dimensions `(Width, Height)`. To make coordinate crops resolution-independent, we normalize coordinates into ratios `[0.0 to 1.0]`:

- Normalized Bounding Box: `bbox = [x1_ratio, y1_ratio, x2_ratio, y2_ratio]`
- When cropping at 200 DPI:
  `crop_x1 = int(bbox[0] * Rendered_Pixel_Width)`
  `crop_y1 = int(bbox[1] * Rendered_Pixel_Height)`
  `crop_x2 = int(bbox[2] * Rendered_Pixel_Width)`
  `crop_y2 = int(bbox[3] * Rendered_Pixel_Height)`

We then add a 15-pixel padding buffer to ensure context (labels and unit metrics like mg/dL or cells/cu.mm) is preserved, and crop using Pillow's `image.crop((x1, y1, x2, y2))`."


Q8: "How is the 100 Ground-Truth Q&A dataset structured?"
----------------------------------------------------------------------------------------------------
ANSWER:
"The dataset is organized into 10 key medical modules in `src/qa_engine.py`:
1. Patient Demographics (IDs 1-5): Name, Age, Gender, DOB.
2. Insurance Policy Header (IDs 6-15): Application No U100723465AD0, Tata AIA.
3. Service Details (IDs 21-25): Service Type - Home Visit.
4. Face Verification (IDs 26-35, 91-95): Similarity 98.75%, FRS 98.75, Distance 0 km, Timestamp.
5. Complete Blood Count (IDs 36-60): Hb 14.92, WBC 7,900, Platelets 2,90,000, RBC 5.88M, ESR 14.
6. Blood Chemistry (IDs 61-70): BUN 18.10 mg/dL, Creatinine 0.88 mg/dL, RBS 112.12 mg/dL.
7. HbA1c Pathology (IDs 71-80): 5.1% (Normal Glucose Control: Yes).
8. Viral Serology (IDs 81-90): HIV 1 & 2 Negative, HBsAg Non-reactive.
9. LFT & Lipid Profile: Bilirubin 0.73, SGOT 23.24, SGPT 24.72, Cholesterol 158.
10. Summaries (IDs 96-100): Face Match, CBC, Serology, Insurance Exam & Full PDF Brief Summary."
